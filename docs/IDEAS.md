# IDEAS

This file holds product ideas worth keeping, without forcing them into immediate implementation.

## Ecosystem Ideas

- Let a mission own or reference a place, a list, a note, and an incident
- Make related objects feel navigable, not just linked
- Add `Opened from ...` context to preserve user orientation
- Offer `Create note for mission`, `Create list for incident`, and similar smart actions

## Home Ideas

- Continue improving customizable widgets
- Keep widget layout simple but flexible
- Make the Home dashboard feel curated, not crowded
- Ensure quick actions reflect visible features only

## People / Access Control Ideas

- Fully editable space roles
- Fully editable visibility groups per space
- Safe guards against removing the last effective admin
- Simple visibility rules like `Everyone`, `Selected groups`, `Only me`

## Budget Ideas

- Better category insights
- Subscription-focused summary
- Category budgets with clearer progress
- Smarter goal planning and progress tracking

## Settings Ideas

- Group settings into intuitive sections
- Add permissions overview
- Add region options: currency, units, locale-sensitive formatting
- Add appearance mode: system, light, dark
- Add premium / subscription management
- Add sync diagnostics or advanced troubleshooting section

## macOS Ideas

- Use split views where they genuinely help
- Avoid forcing iOS sheets into awkward macOS patterns
- Prefer separate platform containers when behavior diverges too much

## UX Heuristics

- Fewer controls, but more useful ones
- Prefer context-aware actions over generic utility menus
- Use native layout and motion as much as possible
- Avoid exposing raw IDs or backend concepts in UI
